From Mobile to MEAN: The Original Android Inventory App

This native Android inventory application was built using Java in Android Studio during my CS360 course. Its purpose was to help small business managers track inventory on the go. The app included secure user authentication, the ability to add/edit/delete items, and a notification system for low-stock alerts.

I selected this artifact because it represents the starting point of my journey in building real-world applications. It was the first project where I implemented mobile UI design, user-focused workflows, and basic security. It highlights my understanding of the mobile development lifecycle, from activity and intent management to persistent local storage.

As I reviewed the code, I identified opportunities for improvement in scalability, maintainability, and platform independence. Modernized Inventory Management: MEAN Stack Web Application

This artifact is an enhanced version of my original Android app, redesigned from the ground up as a modern full-stack web application using MongoDB, Express, Angular, and Node.js. I created this artifact during my CS499 capstone to showcase my growth across all core areas: software engineering, data structures and algorithms, and database development.

I selected this as the enhanced artifact because it exemplifies how I transformed a single-platform mobile app into a scalable, maintainable, secure system accessible from any device. The frontend was redesigned with Angular Material for responsive UI, while the backend leverages RESTful APIs and secure JWT-based authentication for role-based access.

Enhancements included integrating a min-heap priority queue algorithm to surface high-priority restock items and real-time inventory dashboards. I also introduced validation middleware in Node.js and Mongoose schema rules to improve data security and quality.

This project helped me meet course outcomes related to software architecture, real-world algorithmic application, collaboration (through GitHub issue management), and secure development practices. I also recorded a code review video to explain the enhancements and design decisions to peers and reviewers.
